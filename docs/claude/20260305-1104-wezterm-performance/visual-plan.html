Implementation Plan

WezTerm Configuration Performance Improvements

Eliminate the 2-second UI freeze when opening scrollback in nvim, optimize hyperlink array construction, and audit font fallbacks — all within config.lua

Phase 1: Scrollback Handler (Critical) Phase 2: Hyperlink Array (Code Quality) Phase 3: Font Audit (Exploratory)

The Problem

Side-by-side: current behavior vs. desired behavior

Before

  1. User presses Ctrl+Shift+E
  2. Handler calls get_lines_as_text(2000) — physical lines, capped at 2000
  3. Writes temp file, spawns nvim in new window
  4. UI freezes for 2000ms while sleep_ms(2000) blocks
  5. Temp file deleted, UI unfreezes
  6. Hyperlink arrays built with double iteration
  7. 8 font fallbacks, some potentially redundant

After

  1. User presses Ctrl+Shift+E
  2. Handler calls get_logical_lines_as_text(scrollback_rows) — full scrollback, logical lines
  3. Writes temp file, spawns nvim in new window (no sleep)
  4. Zero UI freeze — no sleep_ms, temp file left for OS cleanup
  5. Cursor at bottom via +normal G
  6. Hyperlink arrays built in single pass
  7. Font fallbacks trimmed to only needed fonts

Modified Functions

Three changes, all within config.lua

Phase 1 — Improve Scrollback Handler
config.lua • Improve event handler (lines 95–114)
wezterm.on("trigger-nvim-with-scrollback", function(window, pane) local scrollback = pane:get_logical_lines_as_text( pane:get_dimensions().scrollback_rows) local name = os.tmpname() local f = io.open(name, "w+") if f ~= nil then f:write(scrollback) f:flush() f:close() end window:perform_action( wezterm.action({ SpawnCommandInNewWindow = { args = { "nvim", "+normal G", name }, }, }), pane) -- no sleep, no os.remove — OS cleans /tmp end)
Phase 2 — Code Quality
config.lua — lines 48–78 • Hyperlink array construction
local hyperlink_rules = wezterm.default_hyperlink_rules() local hyperlink_regexes = {} -- Extract regexes from default rules first for _, rule in ipairs(hyperlink_rules) do table.insert(hyperlink_regexes, rule.regex) end -- Add custom rules (build both arrays together) for _, rule in ipairs({ { regex = [[...IP pattern...]], format = "$0" }, { regex = "...localhost pattern...", format = "$0" }, }) do table.insert(hyperlink_rules, rule) table.insert(hyperlink_regexes, rule.regex) -- now built together end

WezTerm APIs Used

Existing APIs referenced in this plan

API Call Parameters Behavior
pane:get_logical_lines_as_text(nlines) nlines — number of logical lines Returns scrollback text respecting soft wraps. Logical lines = actual lines as displayed, not physical line breaks.
pane:get_dimensions() none Returns a table with terminal dimensions including .scrollback_rows for total scrollback size.
wezterm.default_hyperlink_rules() none Returns the built-in set of hyperlink detection rules (URLs, file paths, etc.)
wezterm.font_with_fallback(fonts) fonts — table of font specs Returns a font config that tries each font in order for glyph resolution.
wezterm ls-fonts --text "glyph" glyph string CLI tool: shows which fallback font provides the given glyph. Used for Phase 3 audit.

Edge Cases

Scenarios requiring careful consideration

Scenario Expected Behavior Risk
Large scrollback (10,000+ lines) get_logical_lines_as_text(scrollback_rows) captures full buffer. Temp file write is fast for typical sizes. Low
System under heavy CPU load No sleep or timing dependency. Handler writes file and spawns nvim immediately. None
Empty scrollback (no output) Nvim opens with empty file. Same user experience. None
Missing local.lua pcall catches error, defaults used. No hyperlink rules from local config. None — existing defensive pattern
Font removed in Phase 3 but glyph needed Glyph renders as placeholder box. Visible regression. Medium — mitigated by thorough audit before removal
Custom hyperlink rules from local.lua Still appended to both arrays after refactoring. Behavior unchanged. None
Concurrent Ctrl+Shift+E invocations Each call gets a unique os.tmpname(). No file conflicts. None

Implementation Notes

Backward Compatibility: All changes are internal optimizations. User-facing behavior (keybindings, hyperlink detection, font rendering) remains identical. No configuration format changes. The pcall pattern for local.lua is preserved.
Critical — Handler Changes: The event handler is streamlined: sleep_ms(2000) and os.remove() are removed, get_lines_as_text(2000) is replaced with get_logical_lines_as_text(scrollback_rows), and nvim gets +normal G for cursor-at-bottom. The io and os imports are retained (still needed).
Performance Targets: Scrollback handler: zero UI freeze (down from 2000ms). Startup time <50ms (no regression). Font rendering: no noticeable delay. Hyperlink construction: ~1-2ms improvement (negligible but cleaner code).
