WezTerm Config Architecture ~/.dotfiles/wezterm/

config.lua
Main config: fonts, colors, keybinds, hyperlinks, scrollback handler. 219 lines of Lua.
local.lua
Optional machine-local overrides: font, decorations, URL transforms, hyperlink rules, apply_to_config().
colors/*.toml
Custom color schemes. Ships Tokyo Night Storm (unused — Catppuccin Mocha is active).
install.sh
Symlinks config.lua → ~/.config/wezterm/wezterm.lua and color files.
maps to ghostty key-value config

Hyperlinks & URLs

URL detection partial
Custom regex rules for:
- numeric IPs (127.0.0.1:8000)
- localhost URLs
- local.lua custom patterns
→
link-url = true (default)
Built-in detection only. No custom regex.
URL transforms none
url_transforms: rewrite URLs
before opening (prefix matching)
→
Not supported. No URL rewriting.
Features requiring external tooling: The scrollback-in-nvim handler, regex search with pre-filled pattern, QuickSelect with URL transforms, and custom hyperlink rules all rely on WezTerm's Lua scripting. Ghostty is intentionally configuration-only (no scripting). These features would need shell scripts, tmux integration, or other external tools to replicate.
