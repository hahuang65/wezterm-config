Feature Plan

Open Filepath

Extend Ctrl-Shift-O to handle both URLs and filepaths. URLs open in the browser. Filepaths are resolved and opened with a user-specified command.

config.lua local.example.lua Ctrl-Shift-O QuickSelectArgs PromptInputLine
The Problem
Before
1.User sees a filepath in terminal output (e.g. src/main.rs:42)
2.Ctrl-Shift-O only matches URLs — filepath is not highlighted
3.User must manually copy the path, type a command, and paste
4.Relative paths require mentally resolving against CWD
After
1.User sees a filepath in terminal output
2.Ctrl-Shift-O highlights both URLs and filepaths
3.User selects filepath → prompted for command (e.g. nvim)
4.Path auto-resolved (relative, ~/, $VAR), line number passed to editor

New Functions
config.lua — after starts_with()
local function is_url(str)
  return str:match("^%w+://") ~= nil
end

Detects http://, https://, ftp://, etc. Used to branch URL vs filepath in the callback.

config.lua — parse_filepath()
local function parse_filepath(raw)
  -- Try :line:col first, then :line, then bare path
  local path, line, col = raw:match("^(.+):(%d+):(%d+)$")
  if path then return path, tonumber(line), tonumber(col) end
  path, line = raw:match("^(.+):(%d+)$")
  if path then return path, tonumber(line), nil end
  return raw, nil, nil
end

Extracts path, line, col from strings like src/main.rs:42:10.

config.lua — resolve_filepath()
local function resolve_filepath(path, cwd)
  if path:sub(1, 1) == "~" then         -- ~/...  → $HOME/...
    local home = os.getenv("HOME") or ""
    return home .. path:sub(2)
  end
  local env_var = path:match("^%$(%w+)") -- $VAR/... → expand
  if env_var then
    local val = os.getenv(env_var)
    if val then return val .. path:sub(#env_var + 2) end
  end
  if path:sub(1, 1) == "/" then return path end  -- absolute
  if cwd then return cwd .. "/" .. path end       -- relative
  return path
end

Resolution order: ~ → $ENV_VAR → absolute → CWD-relative.

Regex Patterns
PatternMatchesExamples
/[\w.\-]+(?:/[\w.\-]+)+ Absolute Unix paths /tmp/test.txt, /home/user/.config/nvim/init.lua:10
\.\.?/[\w.\-]+(?:/[\w.\-]+)* Relative ./ and ../ ./src/main.rs, ../config.lua:42:10
~/[\w.\-]+(?:/[\w.\-]+)* Home-relative ~/Documents/notes.md
\$\w+/[\w.\-]+(?:/[\w.\-]+)* Env variable prefix $HOME/.bashrc, $XDG_CONFIG_DIR/nvim
[\w.\-]+(?:/[\w.\-]+)*\.[\w]+ Bare paths with extension src/main.rs, config.lua:5

All patterns include optional (?::(\d+))?(?::(\d+))? suffix for :line:col. Word boundaries (\b) prevent partial matches.

Backward Compatibility. Existing URL behavior is fully preserved. URLs are detected first via is_url(), then via url_transforms prefix matching. Only non-URL, non-transform selections fall through to the filepath handler.
Regex False Positives. The bare-path-with-extension pattern (src/main.rs) may match non-path text like v1.2. This is acceptable — the user can always press Escape at the prompt to cancel.
Line Number Support. +line is only injected for known editors: nvim, vim, vi, nano, code, emacs. Other commands receive just the filepath to avoid breaking their argument parsing.
